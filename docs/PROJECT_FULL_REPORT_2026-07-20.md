## 2. رحلة المشروع — ما أُنجز

### المرحلة الأولى: التحصين الإنتاجي (P6 → P7)
- **P6:** إصلاحات أمنية شاملة (RBAC على كل المسارات، فحوص الملكية، سير اعتماد السائقين مع سجلّ تدقيق، تدوير refresh tokens مع الإلغاء الفوري).
- **P7-01 → P7-06:** بنية تشغيل مؤسسية كاملة: Docker مُقسّى (multi-stage، non-root، read-only، موقَّع)، حزمة compose من 4 خدمات (backend + PostgreSQL + Redis + nginx TLS)، مراقبة كاملة (Prometheus + Grafana + تنبيهات)، نسخ احتياطي مشفَّر مع اختبار استعادة تلقائي (RPO ≤ 15 دقيقة)، وCI/CD بنشر تدريجي وتراجع تلقائي.

### المرحلة الثانية: العمارة المؤسسية (الوثائق)
- **مدونة معمارية كاملة:** G0.0 + ADR-001 حتى ADR-015 + تعديلان — من النموذج المجالي العالمي حتى دستور المنصة.
- **ADR-001 (إصلاح C-1 المالي): مُصادَق عليه ومُنفَّذ** — أخطر عيب في المنصة (تضارب الدفع عند إكمال رحلتين متزامنتين) أُغلق نهائياً: الإكمال + الدفع صارا وحدة ذرّية واحدة، مثبتة باختبارات سباق حية.

### المرحلة الثالثة: الهجرة إلى العمارة النظيفة (13 مرحلة — مكتملة)

## 4. ما تبقّى (بصراحة كاملة)

1. **بوابة PostgreSQL الحية** — محرّك Postgres مكتمل البناء خلف `DB_ENGINE=postgres` مع مترجم لهجة SQL مُختبَر (9 اختبارات وحدة)، لكن **البوابة الأخيرة لم تُنفَّذ**: تشغيل الحصادات ضد Postgres حقيقي في بيئة staging. بيئة الفحص هنا لا تملك خادم Postgres — لم يُزيَّف الاختبار، والتقرير يقولها صراحة.
2. **الالتزام في git** — يوجد **123 ملفاً غير مُلتزَم** يمثل كل إنجاز الهجرة. آخر commit سابق لكل المراحل الـ13. **هذه أعلى مخاطرة حالية**: كل هذا العمل غير محمي بنقطة تراجع في git. التوصية الأولى: commit فوري كخط أساس.
3. **تنظيف المسارات القديمة** — 10 ملفات routes قديمة باقية عمداً كأهداف تراجع؛ تُزال بعد فترة مراقبة إنتاجية.
4. **`npm ci` في بيئة الفحص** يفشل على sqlite3 لسبب شبكي بحت (proxy) — لا علاقة له بالكود؛ يعمل في CI/Docker الطبيعي.
5. **ديون مُسجَّلة معروفة:** رسائل عربية داخل الكود (تنتظر كتالوجات الترجمة)، مخازن rate-limit في الذاكرة (تنتظر Redis)، الهاتف كهوية (مُخطَّط معالجته في خارطة الطريق).
